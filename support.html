COMMON ISSUES

Before you contact us

Most issues are solved by one of these:

💡 Can I resize the app windows?
Yes — every window in ShortsMinter can be resized or maximized. If anything feels cut off or hard to read, simply drag the window edges to resize it or click the maximize button in the top right corner to go full screen. This applies to the main app window and all popup windows including Settings, Captions editor, and the license screen.
🔴 App won't launch after install
Make sure the ShortsMinter Runtime installed successfully. Open your Start Menu and look for "Masondogg Studios" — if the runtime is missing, download the full installer again from your Gumroad confirmation email and run it again.
🔴 Windows shows "Unknown Publisher" warning
This is normal for new independent software. Click "More info" on the blue SmartScreen screen, then click "Run anyway." ShortsMinter is completely safe — this warning disappears as the software builds reputation with Microsoft.
🔴 License key not working
Make sure you are copying the key exactly as it appears in your Gumroad email — no extra spaces. If the key still fails, visit gumroad.com/library to find your purchase and retrieve your key. If you are still having issues use the bug report form below.
🔴 Lost my license key
Check your Gumroad confirmation email first. If you can't find it, log into gumroad.com/library — all your purchases and license keys are stored there. Still can't find it? Use the support form below and we'll help.
🔴 FFmpeg not found error
FFmpeg is bundled in the ShortsMinter Runtime. If you see this error, try restarting your PC after install — this ensures the system PATH is updated correctly. If the error persists after restart, reinstall the runtime from your Gumroad download.
🔴 WhisperX not working or showing ✗
WhisperX requires the full runtime to be installed. Confirm the runtime is installed by checking Start Menu → Masondogg Studios → ShortsMinter Runtime. If missing, reinstall from your Gumroad download. A restart after install may also be required.
🔴 Export failed or video not processing
Check that your source video is a supported format (MP4, MOV, MKV). Make sure your output folder exists and you have write permissions. If the issue persists, copy your App Log from Settings → Support → Copy App Log and include it in a bug report.
💡 Why do caption and transcription results vary?
Transcription quality and processing speed are directly influenced by your PC hardware — this is completely normal and expected across all hardware levels. Here is how each component plays a role:

Storage (biggest impact on speed):
  → HDD (spinning disk) — slowest. Model files load noticeably slower, especially on first run.
  → Standard SATA SSD — significantly faster. Good experience across all Whisper models.
  → NVMe M.2 SSD — fastest available. Models load nearly instantly, storage is no longer a bottleneck.

GPU and VRAM:
  → No NVIDIA GPU — transcription falls back to CPU only (much slower).
  → 4 GB VRAM — limited to smaller Whisper models (tiny, base).
  → 8 GB+ VRAM — can run medium model for noticeably better accuracy.
  → 10 GB+ VRAM — can run large model for best possible accuracy.

GPU Drivers:
  → Outdated NVIDIA drivers can cause CUDA errors or silent fallback to CPU.
  → Always keep NVIDIA drivers up to date for best results.

CPU:
  → More cores and faster clock speed improve transcription when running on CPU.

Getting the best results regardless of hardware:
  → Generate more clips than you need and pick the best ones.
  → Use manual timestamps to target specific moments in your video.
  → Use the Caption Editor in the app to fix any misheard words before exporting.
  → Use the Emoji Editor after export to fine-tune emoji assignments per segment.
  → Try a higher Whisper model in AI Settings if your GPU has enough VRAM.

🔴 Windows Firewall blocking ShortsMinter
On first launch Windows Firewall may block ShortsMinter from accessing the internet. Follow these steps exactly:

1. Close ShortsMinter completely first — firewall changes do not apply to a running app.
2. Open Windows Security → Firewall & network protection → Allow an app through firewall.
3. Click Change settings → Allow another app → Browse → navigate to your ShortsMinter install folder. The default location is C:\Program Files\Masondogg Studios\ShortsMinter\ — select ShortsMinter.exe and click Add. Note: do not browse to the desktop shortcut — you need the actual .exe in Program Files. If you are unsure of your install path: right-click the ShortsMinter desktop shortcut → Properties → copy the path from the Target field.
4. Make sure both Private and Public boxes are checked for ShortsMinter.
5. Click OK to save.
6. Reopen ShortsMinter and enter your license key.

Important: always close the app fully before saving firewall settings, then reopen it fresh.
🔴 McAfee, Norton, Bitdefender or other security software blocking ShortsMinter
Third party antivirus and security suites sometimes block new independent software from making network connections. ShortsMinter is completely safe — it only connects to Gumroad to validate your license and GitHub to check for updates. To fix: open your security software and add ShortsMinter.exe as a trusted application or exclusion. The exact steps vary by software — search for "add exclusion" or "add trusted app" in your security software's help. Once added, relaunch ShortsMinter and activate your license normally.
💡 Why does ShortsMinter require administrator permission to install?
ShortsMinter requires administrator permission to install its AI transcription engine, video processing components, and shared runtime libraries. This is a one-time requirement during installation only and is standard practice for professional video and AI software. The app itself runs normally after installation without requiring ongoing administrator access.
💡 Can I use ShortsMinter in a virtual machine?
ShortsMinter is designed and optimized for physical PC hardware. Virtual machine use is permitted for legitimate purposes such as testing. However, performance will be reduced — GPU acceleration, NVENC encoding, and CUDA-based transcription require direct hardware access that is typically unavailable in virtual machines. CPU-only mode will be used instead.

No guarantee of functionality in VMs: ShortsMinter's ability to function inside a virtual machine depends entirely on your VM software, host operating system, and allocated resources. Factors such as virtualized GPU access, CPU thread allocation, memory limits, and network configuration vary widely between VM setups. We cannot guarantee ShortsMinter will work correctly — or at all — inside a virtual machine. ShortsMinter is built to communicate directly with your physical CPU and GPU for full performance. If you experience issues inside a VM, the recommended solution is to install and run ShortsMinter on your host machine.

GPU passthrough: Even when GPU passthrough is configured, it is not guaranteed to function as intended. NVENC hardware encoding and CUDA-based AI transcription depend on low-level driver access that passthrough implementations do not always fully replicate. Results vary significantly by hypervisor, GPU model, and driver version — and in some configurations passthrough may not work at all. We cannot provide support for VM GPU passthrough configurations.

Important: Using ShortsMinter to duplicate licenses, clone activations, or bypass security in a virtual machine will result in permanent license revocation without refund. All virtual machine activations are logged in our system.
🔴 License activation fails with a security certificate error
This can happen on fresh Windows installations where security certificates have not fully synced yet. The ShortsMinter installer automatically handles this during setup. If you see this error:

1. Wait 2 minutes and click Retry on the activation screen.
2. If retry fails, restart your PC and try again.
3. If the issue persists, open CMD as administrator and run: certutil -generateSSTFromWU roots.sst
4. Restart your PC and try activating again.

This is a Windows system issue — not a problem with your license key.
🔴 Using a VPN and license activation fails
Some VPN configurations block connections to Gumroad's license validation servers. If you are using a VPN and activation fails:

1. Temporarily disable your VPN.
2. Activate your license.
3. Re-enable your VPN after successful activation.

After your first successful activation ShortsMinter caches your license locally for 72 hours so you can work offline or with your VPN enabled.
CREDENTIAL MANAGER

Windows Credential Manager — Required for ShortsMinter

ShortsMinter uses Windows Credential Manager to securely store your license key, API keys, and session data. If it is unavailable on your machine, activation will not work — full stop.

🔑 What it is and why ShortsMinter needs it

Windows Credential Manager is a built-in Windows security vault that stores passwords and license data for applications. ShortsMinter uses it to save your license key, Claude API key, OpenAI API key, and HuggingFace token between sessions — securely, without writing them to any file. If Credential Manager is unavailable, ShortsMinter cannot store or retrieve any of this data. Activation will fail and API keys entered in Settings will disappear every time the app restarts.

💡 How to check if it is enabled on your PC

Click the Windows Start button and search for Credential Manager. It should open a Control Panel window showing "Web Credentials" and "Windows Credentials" tabs. If nothing opens or you get an error, it may be disabled on your machine.

Microsoft's official guide: Accessing Credential Manager on Windows →

🔴 Common causes and how to fix them

1. Service running but set to Manual startup
This is the most common cause — especially on VMs and debloated Windows installs. The service is technically running but was never configured to start automatically. To fix:
  1. Press Windows + R → type services.msc → press Enter
  2. Scroll down to Credential Manager
  3. Right-click → Properties
  4. Change Startup type to Automatic
  5. Click Apply → OK → close the window
  6. Fully exit ShortsMinter — close it completely
  7. Reboot your PC
  8. Relaunch ShortsMinter and activate normally

Note: If the service is already running you do not need to stop and restart it — changing to Automatic and rebooting is enough.

2. Service disabled entirely
Popular "debloater" scripts and optimization tools sometimes disable the Windows Credential Manager service (called VaultSvc) without warning. To re-enable it:
  1. Press Windows + R → type services.msc → press Enter
  2. Scroll down to Credential Manager
  3. Right-click → Properties → set Startup type to Automatic
  4. Click Start → OK
  5. Reboot your PC and relaunch ShortsMinter

3. Security software blocking it
Antivirus or security suites (Bitdefender, Kaspersky, some Norton configs) can block Credential Manager API calls as a security policy. Open your security software and add ShortsMinter.exe as a trusted application or exclusion, then relaunch ShortsMinter.

4. App needs administrator permission
ShortsMinter will detect this automatically and offer a one-click relaunch as Administrator. If that does not resolve it, right-click the ShortsMinter desktop shortcut and select Run as administrator.

If none of the above resolves the issue, use the bug report form below and include your system report — we will diagnose it fast.

💻 Running ShortsMinter in a Virtual Machine?

Credential Manager issues are more common in VM environments. Before contacting support try these steps in order — they cover the most common VM-specific causes:

Step 1 — Run as Administrator
Right-click the ShortsMinter shortcut → Run as administrator. VMs often restrict credential access for non-elevated processes.

Step 2 — Verify VaultSvc from CMD as Admin
Open Command Prompt as Administrator and run:
  sc query VaultSvc — confirm STATE shows RUNNING
  sc config VaultSvc start= auto — set to Automatic
  net start VaultSvc — start it (harmless if already running)

Step 3 — Test Credential Manager directly
Still in CMD as Admin run:
  cmdkey /generic:SMTest /user:testuser /pass:testpass
  cmdkey /list — confirm SMTest appears
  cmdkey /delete:SMTestGeneric — clean up
If SMTest does not appear in the list, the VM's Credential Manager vault is restricted at the OS level — contact support with your VM software name and version.

Step 4 — Reboot and retest
Fully reboot the VM after making any service changes. Changes to VaultSvc startup type do not take effect until after a reboot.

Note: ShortsMinter is designed and optimized for physical PC hardware. Full functionality in VMs is not guaranteed. See the VM disclaimer section above for details.
